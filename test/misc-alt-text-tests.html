Alt-Text Tests

Alt-text using alt="{image description}", should describe the intent and or context of the image not random details. If the image adds no info, it can be marked decorative using alt="" or role="none". See "Hide Image Alt Text" below.

Notes
  • CSS display:none will not be announced since it is not rendered in DOM vs. role=none that keeps in DOM but removes from AT tree
  • role=none or aria-hidden=true cannot be used on a focussable element or on a parent interactive element
  • longdesc="{long image description}" was deprecated

Expose Image Semantics

alt="{description}"

Image with alt-text content. Note title=.. to add additional info (tooltip does not equal alt-text)

Hide Image Semantics

alt=""

Image without alt-text content, marked as decorative using alt="".

role="none"

Image without alt-text content, exposed to accessibility API but removes semantic meaning (empty element).

role="presentation" (avoid)

Equivalent to role=none. DEPRECATED, use role=none instead.

aria-hidden="true" (avoid)

Image without alt-text content, removed from accessibility API. CAUTION inconsistent browser implementation, recommend avoid for now.

Log of Testing

Tested on Windows 10 with combinations of screen readers Jaws, NVDA, Narrator, ChromeVox and browsers Chrome, Edge, FF, IE11.
